User Guide
Navigation
Searching
There are multiple ways to search in Ampache Mobile. You can just start typing on any list being displayed and it will filter the results.

Also there is the search scene, which will send search requests up to your server and back. This is usually the fastest way to your music if you know what you are looking for.

TIP: Just start typing on the main menu and it will automatically navigate to the search scene.

Lists
Often times lists are very long and it takes forever to scroll to the point you are interested in.

On any list that contains dividers simply tap the black header on the top of the screen and a list will appear allowing you to jump to a specific point in a list.

Short Cuts
Tap and Hold any Artist to view all their songs.

Tap and Hold any Song to Play or Enqueue only that song.

Tap and Hold Recent in the main menu to quickly find recently added songs. Note: This is configurable in the preferences menu.

Playlists
Playlists need to be created on your server to be used by Ampache Mobile.

Preferences
Startup Account
This is setting controls how Ampache Mobile starts up. If you select an Account it will automatically be connected upon loading the App. Always Ask will bring up a menu to allow you to select which account to connect to.

Accounts List
Tap "Add Account" to add a new account.

Tap an existing Account to edit it.

Swipe an existing Account to delete it.

Tap and hold an account rearrange the order.

Account Editor
Account Name
This is the name which will show up on the login screen.

URL
Internet address of the server you are attempting to connect to.

NOTE: This is the same address you would put into a web browser to access your server from the internet.

EXAMPLE: http://myserver.dyndns.org/ampache

List Fetch Size
This configures how many items will be requested at a single time from the server. The lower the number, the faster requests will come back but the higher number of total requests to get all the requested items.

Work Arounds
GStreamer %20: Enable this if streaming works in other forms but not your phone. webOS Bug was introduced in 1.4.1

Audio Stall Detection: Enable this your if music simply stops, Ampache Mobile will detect a stall and attempt to recover.

Apache Timeout: This is the tcp timeout of your webserver. Ampache Mobile will attempt to detect potential timeouts and circumvent them. Improper configuration of this value will result in audio stalls. Default is 300
